# 06_frontend

This folder is the implementation layer for the actual application.

## Purpose
- convert generated UI into reusable components
- organize code by features and shared UI primitives
- maintain separation between domain logic and presentation

## Recommended principles
- feature-first domain organization
- reusable core UI components
- consistent AppShell and RightRail across the product
- use component APIs from `09_component_api`
- use data contracts from `08_data_contracts`
