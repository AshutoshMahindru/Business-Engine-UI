# 05_prototypes

This folder stores generated UI artifacts, review iterations, experiments, and comparison notes.

## Purpose
- Keep a history of generated screens and refinements
- Compare design iterations
- Preserve prototype outputs before code implementation
- Support design reviews and prompt tuning

## Structure
- `packet_1/` Core first-pass screens
- `packet_2/` Expanded screens
- `packet_3/` Additional screens or revised systems
- `experiments/` Alternative directions and prompt experiments
- `comparisons/` Before/after notes and evaluation

## Workflow
1. Generate a screen in Figma Make or HTML
2. Store link/screenshot/export here
3. Add notes on what worked, what failed, what changed
4. Compare v1 vs v2 and record learnings
